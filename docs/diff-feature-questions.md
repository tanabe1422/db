# diff 機能 課題表 / 質問事項

テーブル定義(`*.table.json`)の差分表示機能を設計するにあたって、確定させたい論点をまとめます。
各項目の「回答」欄に記入してください（選択肢の記号でも、自由記述でも可）。推奨案には ★ を付けています。

前提（現状把握）:

- バックエンドは Wails(Go)。アクティブディレクトリは常に 1 つで、`scanner.Scan` が `*.table.json` を再帰走査して `TreeNode` ツリーを返す。
- フロントはサイドバーのツリーからファイルを選ぶとタブで `TableDefinitionView`（グリッド）が開く。
- カラムの一意キーは英語列名 `name`。差分対象になりうるカラム属性: `nameJa` / `dataType` / `length` / `precision` / `scale` / `notNull` / `unique` / `pk`(primaryKey 由来) / `defaultValue` / `remarks` / インデックスマーカー。

## B. フォルダ単位の diff

### B-1. 比較する 2 フォルダの範囲（クロスディレクトリ）
現状アクティブディレクトリは 1 つだけです。「2 つのフォルダ」をどこから選びますか。
- (a) ★ 同一アクティブディレクトリ配下のサブフォルダ 2 つを選ぶ。
- (b) 別々のルートディレクトリ（例: 旧バージョン vs 新バージョン）も選べるようにする。→ ディレクトリ選択 UI とバックエンド走査の拡張が必要。
- (c) 任意のフォルダを 2 つダイアログで選ぶ（設定ディレクトリに縛られない）。→ Go 側のパス安全チェック緩和が必要。

回答:

### B-2. ファイルのマッチング基準
左右フォルダのどのファイル同士を「同じファイル」とみなすか。
- (a) ★ フォルダからの相対パスが一致するもの（サブフォルダ構造込み）。
- (b) ファイル名（basename）だけで一致。サブフォルダ構造は無視。

回答:

### B-3. 再帰の有無
- (a) ★ サブフォルダも再帰的に含めて比較。
- (b) 直下のファイルのみ比較。

回答:

### B-4. ファイル一覧 diff の表示内容
左右にファイル一覧を表示する際、各ファイルの状態をどう示すか。
- 状態: 変更あり / 追加（右のみ） / 削除（左のみ） / 変更なし。
- (a) ★ 左右 2 カラムで対応行を揃えて表示。追加=右行を緑、削除=左行を赤、変更=黄、一致=通常色。各行にステータスアイコン。
- (b) 1 リストにまとめてステータスバッジを付ける。

回答:

### B-5. 「変更なし」ファイルの扱い
- (a) ★ デフォルト非表示（差分のあるファイルだけ表示）。トグルで表示可。
- (b) 常に全件表示。

回答:

### B-6. ファイル一覧での「変更あり」判定基準
何をもって「このファイルは変わった」とするか。
- (a) ★ 正規化後の意味的な差分（後述 C の比較ロジック）で 1 つでも差分があれば変更あり。
- (b) ファイルのバイト/テキストが違えば変更あり（整形やキー順の違いも変更扱い）。

## C. ファイル単位の diff（テーブル定義）

### C-1. 比較の対象範囲
カラム配列以外に、どこまで差分表示しますか。
- (a) ★ カラムに加えて、テーブルメタ（`name` / `nameJa` / `description` / `primaryKey`）とインデックスも差分表示する。
- (b) まずはカラム配列のみ。メタ/インデックスは対象外（後回し）。

回答:

### C-2. カラムの対応付けキー
- (a) ★ 英語列名 `name` をキーにする（ご要望どおり）。左右で同じ `name` のカラムを突き合わせる。
- (b) name + 位置 など複合。

補足質問: 同じ `name` が左右で**順番だけ**変わった場合（並べ替え）は「変更」とみなしますか？
- (i) ★ 並べ替えは差分としない（内容が同じなら一致扱い。表示は左の順序基準）。
- (ii) 並べ替えも差分として強調する。

回答:

### C-3. セル単位ハイライトの粒度
「変わった場所は黄色」をどの単位で塗るか。
- (a) ★ 変わった**セル（フィールド）単位**で黄色（例: `dataType` だけ変わったらその列だけ黄色）。
- (b) 行全体を黄色。

回答:

### C-4. 差分とみなすフィールド
黄色対象に含めるカラム属性（チェックしたいものを指定）。
- 既定の推奨対象 ★: `nameJa` / `dataType` / `length` / `precision` / `scale` / `notNull` / `unique` / `pk` / `defaultValue` / `remarks`
- インデックスマーカー（どのインデックスに参加しているか）も差分対象にしますか？
  - (i) ★ 含める（C-1 でインデックスを対象にする場合）。
  - (ii) 含めない。

回答:

### C-5. 行レベルの色（追加/削除）
- (a) ★ 右のみに存在する行 = 右側を緑背景、左のみに存在する行 = 左側を赤背景（ご要望どおり）。左右整列で、無い側は空行プレースホルダ。
- (b) 別案があれば記入。

回答:

### C-6. 値比較の正規化
省略可能な項目の扱い（例: `notNull` 未指定 = false、`defaultValue` 未指定 と 空 など）。
- (a) ★ 正規化して意味的に比較（`notNull:false` と 未指定 は同じ＝差分なし）。
- (b) JSON の生の値で厳密比較。

回答:

### C-7. ファイル diff の表示形式
- (a) ★ 既存の編集グリッドと同じ列構成で、左右に 2 つのグリッドを並べる（横スクロール同期）。
- (b) 1 つのグリッドに左右の値を併記（セル内に「旧→新」）。
